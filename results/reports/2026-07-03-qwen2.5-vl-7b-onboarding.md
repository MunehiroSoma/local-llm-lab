
## Layer 3 Standard Benchmark

Not run in this PR.

Reason: the repository currently has an OpenAI-compatible Layer 3 path for text benchmarks and a deterministic
screenshot scorer for custom VLM tasks, but no fixed MLX-VLM standard benchmark runner for MMMU / VLMEvalKit /
llm-jp-eval-mm. Recording a synthetic screenshot task as Layer 3 would blur the already documented split between
standard benchmarks and custom VLM evaluation.

This is recorded as a benchmark-infrastructure gap, not as a model runtime failure.

## Verdict Proposal

Proposal: `hold`.

The model fits and has usable generation speed on Mac/MLX-VLM. However, its custom screenshot score is below the
existing Gemma4 screenshot baseline (`6 / 10`, `task_score=0.72` vs Gemma4 `10 / 10`, `task_score=0.98`), and Layer 3
standard VLM benchmark infrastructure is not fixed in this repo yet.

Keep `qwen2.5-vl-7b` as a Mac VLM baseline candidate, but do not adopt it as the preferred screenshot/OCR baseline.
