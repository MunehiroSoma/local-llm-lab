# 観点: データ操作・リソース管理

> [`review`](../../../.claude/skills/review/SKILL.md) スキルの観点別サブエージェントが読む単位。
> 入力範囲: **差分のみ**（既存実装への指摘はしない）。
> 想定モデル: 上位モデル（データ整合性の判断が必要なため）。
> 旧 review スキルの「データ操作」観点が ADR 0006 の観点分割時に脱落していたため復活（Issue #46）。

| No | チェック項目 | 区分 | 重要度 | 備考 |
|---|---|:---:|:---:|---|
| 1 | DB/ファイル操作がトランザクション境界内で完結している（部分的な書き込みが残らない） | 手動 | Must | |
| 2 | ループ内で個別クエリ/リクエストを発行する N+1 パターンになっていない | 手動 | Should | 一括取得/JOINで代替可能か確認 |
| 3 | ファイル/DB接続/ロック等のリソースが確実に解放されている（`with`文・`finally`） | 手動 | Must | `results/raw/` の生ログ書き込み・読み込みで特に注意 |
| 4 | 並行処理（マルチプロセス/スレッド/非同期）で共有リソースへの競合が制御されている | 手動 | Should | 並列計測実行時の `results/results.csv` 書き込み競合など |

> `results/results.csv` の追記のみルール自体は `guard_results_append_only.sh`（自動）と
> [`06-harness-eval-registry.md`](06-harness-eval-registry.md) が担当。本観点は一般的なデータ操作の質を見る。
