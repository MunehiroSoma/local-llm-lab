# ADR 0002: インターフェースは OpenAI 互換 API に固定

- ステータス: accepted
- 日付: 2026-07-01

## 背景 / 課題
モデル/ランタイムを頻繁に入れ替える前提で、アプリ・評価ハーネスをどう疎結合にするか。

## 決定
**アプリ/エージェント/評価ハーネスは常に OpenAI 互換エンドポイントだけを見る**。
裏のランタイム（vLLM / vLLM-Omni / MLX-VLM / Ollama / llama.cpp）やモデルは差し替え自由。
モデルは `registry/models.yaml` で宣言管理（コード直書き禁止）。

## 理由 / 代替案
- 各ランタイムが OpenAI 互換サーバを提供 → クライアント・計測コードを1つに固定でき公平。
- アプリ無変更でモデル交換が可能（将来対応の前提）。

## 影響
research §7.4, §13.5。harness/common は OpenAI 互換クライアントを共通利用。
