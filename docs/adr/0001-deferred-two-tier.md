# ADR 0001: 二段構えは「遅延評価型」を採用

- ステータス: accepted
- 日付: 2026-07-01

## 背景 / 課題
単発用途で「テキスト最速エンジン(ExLlamaV3/SGLang/TRT-LLM)」と「多モーダル(vLLM系)」を
最初から2スタック常設するか、1スタックで始めるか。

## 決定
**遅延評価型の二段構え**。まず全環境を一段（NVIDIA=vLLM/vLLM-Omni、Mac=MLX-VLM、共通基準=Ollama）で
立て、PoCで単発を実測。**コーディング単発が体感不足 かつ 高速エンジンが vLLM比で実測 ≥ 約20–30%
速く安定**な場合に限り、②③へテキスト高速レーンを増設。

## 理由 / 代替案
- 2スタック常設は個人運用で運用コストが速度差に見合わない（EXL3 vs FP8、別ポート、更新追従）。
- vLLMはテキストでも十分速い（70B Q4で ~48 tok/s 実測例）。
- Macは MLX が最速兼多モーダルのため常に一段で完結（二段の論点はNVIDIAのみ）。

## 影響
research §4.5 / §13 Phase 4（条件付き）。
